# dotnet-razor-pages-ssrs-report-viewer
Built a .NET Core Razor Page with an embedded SSRS Report Viewer — and added dynamic parameter controls for a seamless report experience.

What I solved:
• Razor Pages + SSRS integration for server-side report rendering.
• Dynamic Country & Region parameters bound to dropdowns (supports ALL / single selection).
• Date range picker to filter results on the server before rendering.
• Smooth UX: parameter panel + preview button + live report viewer (see screenshot).

Why it matters:
• Faster insights — users can slice reports without leaving the page.
• Cleaner separation — UI handles parameter selection; server returns only filtered report data.
• Reusable pattern — drop this into other Razor Pages that need interactive SSRS reports.

If you’re interested, I can share:

1. the Razor Page + handler code (C#)
2. SSRS report parameter setup and Report server tips
3. a runnable sample project
